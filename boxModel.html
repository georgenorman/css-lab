Box Model

Basic Box Model

The CSS box model represents the space occupied by an element. It consists of four areas: margin, border, padding, and content. The image below shows a <div> with a 10px margin, 20px border, 30px padding and 100px content region.

Position - Absolute

This example uses a set of nested divs, with middleGrid having position:absolute, to avoid issues with Margin Transfer. I'm assuming this is the typical case, because using the default static position is very convoluted (as is shown in the following section).

Position - Static

This example keeps everything the same as the Position - Absolute example above, except the position of middleGrid has been changed from absolute to static. This causes the top margin of innerBox to transfer to the outermost div (as evidenced by the more than 10px gap above the outermost div). This is very similar to what was described in the Positioning lab - See the Margin Transfer section for details. Summary: outermost has no border, padding or margin; middleGrid has a 10px margin and position:static; innerBox has a 10px margin. Current 'position' styles:

Position - Static with Border

Similar to what was found in the Positioning lab, by adding a top border to an outer div, the Margin Transfer "feature" can be blocked. The example below, shows that adding a top border to middleGrid, protects it from the margin transfer. However, outermost is still affected by the margin transfer, as evidenced by the more than 10px gap above the outermost div. Position - Static with Padding

Adding some padding to the outermost div can also mitigate the affects of Margin Transfer, when using static divs. Furthermore, the end result can be made to look exactly like the example that uses position:absolute, but with a lot of finagling of the outer and inner divs.
