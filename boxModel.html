Basic Box Model

Multiple Static Inner Boxes

This experiment is the same as above, except now the multiple innerBoxes are static (static boxes nested inside of absolute boxes). Note: If the innerBoxes were relative positioned, the results would be the same.

The following behaviours of the innerBoxes can be observed:

  • They flow around each other
  • They stretch horizontally, to fill the space of their parent container (middleGrid), unless width or margin are explicitly set as is the case for the second innerBox.
  • The top margin of the lower box overlaps the bottom margin of its above sibling.

Position - Static

This series of experiments uses a set of nested divs, with middleGrid and innerBox having position:static (the default position). This causes issues with Top Margin Transfer.

With Content Area

This experiment includes a single 100px by 100px content area. Since position is now static, the top margin of innerBox is transferred to the outermost div (as evidenced by the more than 10px gap above the outermost div). Summary: outermost has no border, padding or margin; middleGrid has a 10px margin and position:static; innerBox has a 10px margin. Current 'position' styles:

Position - Static with Border

Similar to what was found in the Positioning lab, by adding a top border to an outer div, the Margin Transfer "feature" can be blocked. The example below, shows that adding a top border to middleGrid, protects it from the margin transfer. However, outermost is still affected by the margin transfer, as evidenced by the more than 10px gap above the outermost div. Position - Static with Padding

Adding some padding to the outermost div can also mitigate the affects of Margin Transfer, when using static divs. Furthermore, the end result can be made to look exactly like the example that uses position:absolute, but with a lot of finagling of the outer and inner divs.
